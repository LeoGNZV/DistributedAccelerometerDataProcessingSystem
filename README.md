# DistributedAccelerometerDataProcessingSystem

## Описание архитектуры
Проект представляет собой распределённую систему обработки данных акселерометра на C++.

Состав проекта - три узла: 
1. Сервер
2. Генератор данных (узел A)
3. Обработчик данных (узел B)
Для транспортировки используется TCP IPv4. Формат данных - JSON файлы.

## Описание взаимодействия компонентов
```
1. Server запускается на двух портах 
2. Узел B подключается к Server на один из портов
3. Узел A подключается к Server на другой порт
4. Происходит установка соединений: между узлом A и серверов и между сервером и узлом B
5. Данные акселерометра эмулируются на узле A в виде синусоидального сигнала
5. Данные отсправляются как JSON-пакеты серверу (строка с добавлением '\n' в конце)
6. Сервер принимает JSON от узла A, преобразует в структуру на языке С++ и производит проверку на дупликат
7. Сервер пересылает JSON строку со '\n' узлу B
8. Узел B получает симулированные данные акселерометра
9. Узел B вычисляет модуль вектора ускорения: sqrt(x² + y² + z²)
10. Узел B отправляет результат обратно на сервер
11. Сервер сохраняет результат в файле module.log в папке accel
12. Затем снова делается шаг 5 и далее
```

## Инструкции по сборке
Проект запускается на Ubuntu 22.04
Обновление системы
```
sudo apt update
```
Установка компилятора g++ для языка C++ 
```
sudo apt install -y g++ build-essential
```
Установка CMake
```
sudo apt install -y cmake
```
Установка библиотеки nlohmann для работы с json файлами
```
sudo apt install -y nlohmann-json3-dev
```
Скачивание проекта из удалённого репозитория
```
git clone https://github.com/LeoGNZV/DistributedAccelerometerDataProcessingSystem.git
```
Непосредственная сборка проекта
```
cd DistributedAccelerometerDataProcessingSystem
mkdir build
cmake -S . -B build
cd build
make -j$(nproc)
```
## Команды запуска
Нужно запустить части проекта на трёх разных терминалах в указанном порядке:
1. ./build/server/accel_server <номер порта 1> <номер порта 2>
2. ./build/node_b/node_b <номер порта 2>
3. ./build/node_a/node_a <номер порта 1>
Номер порта может принимать значения от 0 до 65535 (2¹⁶- 1)
## Обоснование выбора формата сериализации JSON в сравнении с альтернативами (MessagePack, CBOR, Cap'n Proto).
JSON формат имеет отличную читаемость и он достаточно прост в отладке, что отличает его на фоне остальных. Размер данных и, соответственно, скорость передачи проигрывают другим вариантам форматов, но в этом проекте объём передаваемой информации не велик и скорость передачи информации не настолько важна: мы не упираемся в то, что данные нужно получать ещё быстрее. Также JSON в качестве простого формата для работы отличают минимум зависимостей и отличная поддержка C++ 17.
